## Prerequisites

Before you begin, make sure you have the following installed:

- **Node.js 18+** and **npm 8+**
- **Bun** (used as the package manager across the monorepo)
- **MongoDB** — a local instance or a connection string from [MongoDB Atlas](https://www.mongodb.com/atlas)
- **Expo CLI** — for mobile development (`npm install -g expo-cli`)
- **Git**

<Note>
  The Mention monorepo uses **Bun** as its package manager. The root `package.json` scripts call `bun` directly (e.g. `bun run --filter '*' dev`). Install Bun from [bun.sh](https://bun.sh) before running any workspace commands.
</Note>

## Setup

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/OxyHQ/Mention.git
    cd Mention
    ```
  </Step>

  <Step title="Install all dependencies">
    Run the root-level install script. This installs dependencies for every workspace package and also builds `@mention/shared-types` via the `postinstall` hook.

    ```bash
    npm run install:all
    ```

    Internally this runs `bun install` at the repo root, which resolves all `packages/*` workspaces in one pass.
  </Step>

  <Step title="Configure the backend environment">
    Create a `.env` file inside `packages/backend/` with the following variables:

    ```env
    # Database
    MONGODB_URI=your_mongodb_connection_string

    # Server
    PORT=3000
    NODE_ENV=development

    # External services
    OPENAI_API_KEY=your_openai_api_key
    TELEGRAM_BOT_TOKEN=your_telegram_bot_token
    ```

    <Tip>
      For local development you can use `MONGODB_URI=mongodb://localhost:27017/mention`. If you prefer Docker, the included `docker-compose.yml` spins up MongoDB 7 and Redis 7 alongside the backend automatically:

      ```bash
      docker compose up
      ```
    </Tip>
  </Step>

  <Step title="Start development servers">
    **All services at once (recommended):**

    ```bash
    npm run dev
    ```

    This runs `bun run --filter '*' dev`, starting every package's `dev` script in parallel.

    **Or start services individually:**

    ```bash
    # Frontend only
    npm run dev:frontend

    # Backend only
    npm run dev:backend

    # Agora (audio/video spaces) only
    npm run dev:agora

    # MCP server only
    npm run dev:mcp
    ```
  </Step>

  <Step title="Open the app">
    **Web browser:**

    The Expo dev server opens in tunnel mode by default. Press `w` in the Expo CLI terminal to open the web version, or navigate to `http://localhost:8081`.

    **iOS / Android via Expo Go:**

    1. Install [Expo Go](https://expo.dev/go) on your device.
    2. Scan the QR code printed in the terminal after `npm run dev:frontend`.
    3. The app loads instantly over the Expo tunnel.

    **Backend API:**

    The Express server listens on `http://localhost:3000`. Check its health at:

    ```bash
    curl http://localhost:3000/health
    ```
  </Step>
</Steps>

## Available Root Scripts

Here is the full list of scripts defined in the root `package.json`:

```json
{
  "dev": "bun run --filter '*' dev",
  "dev:frontend": "bun run --cwd packages/frontend dev",
  "dev:backend": "bun run --cwd packages/backend dev",
  "dev:agora": "bun run --cwd packages/agora dev",
  "dev:mcp": "cd packages/mcp && bun run dev",
  "dev:mcp:http": "cd packages/mcp && bun run dev:http",
  "build": "cd packages/shared-types && bun run build && cd ../backend && bun run build && cd ../mcp && bun run build",
  "build:shared-types": "cd packages/shared-types && bun run build",
  "build:frontend": "cd packages/frontend && bun run build",
  "build:backend": "cd packages/shared-types && bun run build && cd ../backend && bun run build",
  "build:agora": "cd packages/agora && bun run build",
  "test": "bun run test --filter '*'",
  "lint": "bun run lint --filter '*'",
  "clean": "rm -rf node_modules packages/*/node_modules",
  "install:all": "bun install",
  "start:frontend": "cd packages/frontend && bun run start",
  "start:backend": "cd packages/backend && bun run start",
  "start:agora": "cd packages/agora && bun run start",
  "start:mcp": "cd packages/mcp && bun run start",
  "start:mcp:http": "cd packages/mcp && bun run start:http"
}
```
